# Petrol Pump Management System (Dynamic MERN Application)

## Project Overview

Build a full-stack petrol pump management web application using the MERN stack (MongoDB, Express, React, Node.js).

This is an internal system (no public homepage) used by employees.

The system must be **fully dynamic and configurable**, meaning:

* Fuel types, tanks, nozzles, and units are NOT hardcoded
* Admin can create, edit, and delete system structure

---

## Authentication & Roles

Use JWT authentication.

Roles:

* admin
* manager
* pumper

Redirect after login:

* /admin
* /manager
* /pumper

---

## System Design (Dynamic)

### Entities must be configurable:

* Fuel Types
* Tanks
* Nozzles
* Pump Units

---

## Default Initial Setup (Seed Data)

Fuel Types:

* Motospot (Petrol)
* High Speed Diesel
* SP95
* Green Diesel

Tanks:

* 4 tanks (one per fuel type)

Nozzles:

* 12 total:

  * 4 Motospot
  * 4 High Speed Diesel
  * 2 SP95
  * 2 Green Diesel

Pump Units:

* Unit 1 → 4 Motospot nozzles
* Unit 2 → 4 High Speed Diesel nozzles
* Unit 3 → remaining nozzles (SP95 + Green Diesel)

NOTE:
This is only default data. Admin must be able to modify everything.

---

## Admin Capabilities (FULL CONTROL)

Admin can:

* Create / update / delete users
* Create / update / delete fuel types
* Create / update / delete tanks
* Create / update / delete nozzles
* Create / update / delete pump units
* Assign nozzles to units
* Change fuel type of tanks
* Modify system structure at any time

IMPORTANT:
System must not break when configuration changes.

---

## Core Features

### 1. User Management (Admin)

* Add employee
* Remove employee
* Assign roles

---

### 2. Pump Assignment (Manager)

* Assign pumpers to units
* Each pumper only sees their assigned unit

---

### 3. Shift Management

* Start shift
* End shift
* All readings belong to a shift

---

### 4. Nozzle Reading Entry (Pumper)

Inputs:

* opening_reading
* closing_reading
* price_per_litre (manual)

System calculates:

* litres_sold = closing - opening
* total_amount = litres_sold × price

Validation:

* closing > opening
* price > 0

---

### 5. Tank Management

Each tank:

* fuel_type (reference)
* capacity
* current_level

Formula:
current_level = current_level + purchases - sales

Constraints:

* No overflow
* No negative stock

---

### 6. Fuel Purchase (Manager)

Inputs:

* tank_id
* quantity_litres
* price_per_litre (manual)
* total_cost (auto)
* supplier (optional)
* invoice_number (optional)

Effects:

* Increase tank level

---

### 7. Profit & Loss (Admin)

Revenue:

* Sum of sales

Cost:

* Sum of purchases

Profit:

* revenue - cost

Provide:

* Daily report
* Fuel-wise report
